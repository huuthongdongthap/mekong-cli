MISSION: Implement the final expansion module – **Market Expansion & Autonomous Evolution** – to scale the AGI trading system across new assets, chains, and traditional markets, while enabling self-optimization through the Genetic Synthesizer and DAO governance. This module will:

1. **Dynamic Asset Universe Manager**: Automatically adds/removes trading pairs based on liquidity, volatility, and backtested performance.
2. **Cross-Chain & RWA Integration Hub**: Activates Phase 7 (Cross-Chain Arbitrage) and Phase 11 (RWA Oracle) with real on-ramps to traditional markets (simulated for now).
3. **Hardware Acceleration Enablement**: Switches from simulated FPGA to real FPGA if available, and optimizes eBPF/XDP for ultra-low latency.
4. **DAO Governance Activation**: Brings the Phase 8 DAO online with real governance tokens, enabling autonomous treasury management and strategy voting.
5. **Genetic Synthesizer Auto-Promotion**: Allows the Phase 5 engine to automatically promote new strategies to live trading after successful backtesting and canary evaluation.

All code must be in TypeScript with 100% Jest test coverage (mocking hardware, blockchain, and external APIs). Provide a configuration file `config.expansion.json` and integrate with the NextJS dashboard for real-time visibility.

**MODULE: Market Expansion & Evolution Orchestrator**
- File path: `apps/algo-trader/src/expansion/`
- Subcomponents:

  **1. Asset Universe Manager**
  - `liquidityScanner.ts`: Periodically queries exchanges (CEX/DEX) for all available symbols, filters by minimum 24h volume (configurable), and maintains a dynamic list of eligible pairs.
  - `volatilityAnalyzer.ts`: Computes rolling volatility (e.g., 1-hour, 24-hour) for each pair; excludes pairs with too low/high volatility (configurable bounds).
  - `backtestScheduler.ts`: For new pairs that pass filters, automatically runs a short backtest (e.g., 7 days) using the current active strategies. If Sharpe > threshold, adds the pair to the live trading list.
  - `riskAdjuster.ts`: Adjusts position sizing and max exposure for new pairs based on volatility and liquidity.
  - `index.ts`: Main loop that runs daily, updates the asset universe, and reconfigures the trading engine.

  **2. Cross-Chain & RWA Integration Hub**
  - `chainConnector.ts`: Activates and monitors cross-chain bridges (Wormhole, Axelar) for the chains enabled in config (Ethereum, Solana, BSC, etc.). Uses mock bridges in simulation.
  - `rwaOracleManager.ts`: Connects to real RWA oracles (Chainlink, etc.) for assets like tokenized stocks, commodities. Fetches latest prices and checks for deviations.
  - `arbitrageEnabler.ts`: Dynamically enables Phase 7 triangular paths that include DEXs on new chains and RWA tokens when profitability is expected.
  - `complianceChecker.ts`: For RWA, ensures that the system respects any simulated regulatory constraints (e.g., only trade during market hours, respect geographic restrictions).
  - `index.ts`: Initializes connections and periodically updates the arbitrage graph.

  **3. Hardware Acceleration Enablement**
  - `fpgaDetector.ts`: Checks if real FPGA hardware is present (via PCIe device scan or mock). If found, loads the actual FPGA driver (simulated) and switches Phase 10 from simulation to hardware mode.
  - `ebpfOptimizer.ts`: If running on Linux with eBPF support, loads the optimized XDP program for packet injection; otherwise falls back to simulation.
  - `latencyBenchmark.ts`: Continuously measures hardware-accelerated vs. software latency and reports the difference.
  - `index.ts`: Initializes hardware acceleration and provides a status flag to the core engine.

  **4. DAO Governance Activation**
  - `tokenDeployer.ts`: If not already deployed, deploys the governance token contract (ERC20) on a chosen chain (simulated via Hardhat fork). Mints initial supply to the treasury.
  - `treasuryManager.ts`: Manages the on-chain treasury (stablecoins, LP tokens) and executes approved proposals (transfer funds, add liquidity, etc.).
  - `proposalExecutor.ts`: Listens for passed proposals from the DAO contract (via event logs) and executes the corresponding actions (e.g., update risk limits, enable new strategies).
  - `votingPowerTracker.ts`: Calculates voting power based on token holdings and delegates.
  - `index.ts`: Starts the DAO listener and exposes treasury metrics.

  **5. Genetic Synthesizer Auto-Promotion**
  - `strategyEvaluator.ts`: Periodically (e.g., weekly) evaluates the performance of all live strategies. If any strategy underperforms (Sharpe < threshold), it marks it for retirement.
  - `candidatePromoter.ts`: Monitors the Genetic Synthesizer (Phase 5) for new candidate strategies that have completed backtesting and paper trading. If a candidate outperforms the worst live strategy, it promotes it to live trading (hot-swap).
  - `rollbackGuard.ts`: After promotion, closely monitors the new strategy for 24 hours. If it causes losses or instability, automatically rolls back to the previous version.
  - `index.ts`: Runs the evolution loop in coordination with Phase 5.
