The framework must automatically spin up a staging environment using Docker containers (simulated or real), deploy all phase modules, and inject various failure scenarios (network latency, packet loss, process crashes, exchange API errors) to validate system resilience. It should generate detailed reports on recovery time, fallback behavior, and any data corruption. All code must be in TypeScript with 100% Jest test coverage (mocking Docker and network interfaces). **MODULE: Chaos Testing Orchestrator**
- File path: `apps/algo-trader/src/testing/chaos/`
- Subcomponents:
  - `environmentBuilder.ts`: Uses `dockerode` (or mock) to create a Docker Compose environment consisting of:
    - Mock exchange servers (Binance, Bybit, Uniswap) that can simulate latency, errors, and orderbook anomalies.
    - Local blockchain forks (Hardhat for Ethereum, Solana test validator).
    - All 12 phase modules, each running in a separate container with simulated market data.
  - `failureInjector.ts`: Provides methods to inject failures at runtime:

    - Duration
    - Target service(s)
    - Expected recovery behavior (assertions)
  - `monitorAggregator.ts`: Collects metrics from all components during the test:
    - System logs (per container)
    - Order execution success/failure
    - Circuit breaker triggers (Phase 5)
    - Swarm reallocation events (Phase 8)
    - Latency histograms
  - `recoveryValidator.ts`: After each failure, verifies that the system returns to normal operation within a specified time and without data loss. Uses assertions defined in the scenario.
  - `reportGenerator.ts`: Produces an HTML/JSON report with timeline, failures, recovery metrics, and any violations.
  - `index.ts`: Main entry point.
- Tests: Mock Docker and failure injection; verify scheduling and reporting.
