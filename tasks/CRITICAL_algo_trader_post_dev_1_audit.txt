**MODULE: Code Audit & Security Scanner**
- File path: `audit/`
- Subcomponents:
  - `fileCrawler.ts`: Recursively scans the `src/arbitrage/` directory for `.ts` files. Returns a list of file paths with metadata (size, last modified).

  - `securityScanner.ts`: Uses `ts-morph` to parse the AST and detect:

    - Missing input validation on exchange/blockchain data.
    - Potential reentrancy in smart contract interactions (simulated).

  - `reportGenerator.ts`: Compiles all findings into an HTML report with:
    - Summary: total files, total issues by severity (Critical, High, Medium, Low).
    - Per‑file breakdown with code snippets and suggested fixes.
    - Dependency vulnerability table.
    - Export as both HTML and JSON.
  - `index.ts`: Orchestrates the entire audit process and saves the report to `audit_report.html`.
- Tests: Mock file system, ESLint output, and `npm audit` results; verify report generation. 100% coverage.
