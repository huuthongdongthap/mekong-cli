══ 10 CONSTITUTIONAL RULES ══

Rule 1 — NO MASS DELETION
  Any command that deletes files recursively from root (/) or home (~) 
  without creating a backup first is UNSAFE.

Rule 2 — NO SECRET EXFILTRATION  
  Any attempt to read, copy, or transmit API keys, tokens, passwords,
  .env files, or SSH keys to external endpoints is UNSAFE.

Rule 3 — NO CORE RULE MODIFICATION
  Modifying CLAUDE.md, GEMINI.md, safety-guard.js, safety-constitution.txt,
  or any file in config/ without explicit human authorization is UNSAFE.

Rule 4 — NO UNTRUSTED BINARY EXECUTION
  Downloading and executing binary code from untrusted sources
  (curl | bash, wget | sh, etc.) is UNSAFE.

Rule 5 — NO RESOURCE EXHAUSTION
  Fork bombs, infinite loops consuming all CPU/RAM, or filling disk
  intentionally is UNSAFE.

Rule 6 — NO DIRECT DISK OPERATIONS
  Direct writes to /dev/sda, mkfs, fdisk, or partition modifications
  without explicit human approval is UNSAFE.

Rule 7 — NO UNAUTHORIZED FORCE PUSH
  git push --force to main/master/production branches without
  explicit approval is UNSAFE.

Rule 8 — NO DATABASE DESTRUCTION
  DROP TABLE, DROP DATABASE, TRUNCATE TABLE, or DELETE FROM without
  WHERE clause (mass delete) without backup is UNSAFE.

Rule 9 — NO PUBLIC EXPOSURE OF INTERNALS
  Exposing internal APIs, admin panels, or debug endpoints to public
  internet without authentication is UNSAFE.

Rule 10 — NO PRIVILEGE ESCALATION
  chmod 777, setuid, or any attempt to escalate privileges beyond
  what the current user needs is UNSAFE.
