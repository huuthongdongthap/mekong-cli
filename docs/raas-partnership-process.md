## Overview

### What is RaaS Partnership?

RaaS Partnership enables agencies, consultancies, and system integrators to resell AgencyOS Revenue-as-a-Service to their clients while earning recurring revenue and accessing the community fund.

### The Partner Value Chain

```
┌─────────────────────────────────────────────────────────────────┐
│  Partner signs client → Client uses RaaS → Revenue generated   │
└────────────────────┬────────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────────┐
│  Revenue Split (6-Way)                                          │
│  ├── Platform (AgencyOS): 20%                                   │
│  ├── Expert (Partner): 30%                                      │
│  ├── AI Compute: 15%                                            │
│  ├── Developer Fund: 15%                                        │
│  ├── Community Fund: 10% ──┐                                    │
│  └── Customer Reward: 10%  │                                    │
└─────────────────────────────┼────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Community Fund → Quadratic Funding → More Matches → Revenue   │
│  (Virtuous Cycle)                                               │
└─────────────────────────────────────────────────────────────────┘
```

### Why Partner?

| Benefit | Description |
|---------|-------------|
| **Recurring Revenue** | 30% of every client's subscription, forever |
| **No Delivery Work** | AI agents execute missions — you focus on strategy |
| **Community Fund Access** | Apply for quadratic funding for client projects |
| **White-Label Option** | Brand RaaS as your own (Enterprise partners) |
| **Lead Sharing** | Enterprise inbound leads routed to certified partners |

## Community Fund Participation

### What is the Community Fund?

The Community Fund is a pooled resource (10% of all RaaS revenue) that funds projects benefiting the RaaS ecosystem through **quadratic funding**.

### Quadratic Funding Formula

```
matched = (Σ√ci)² - Σci

Where:
- ci = individual contribution i
- Σ√ci = sum of square roots of contributions
- matched = matching amount from Community Fund
```

**Simple Example:**
```
Project: "Add OAuth2 integration for Shopify"

Contributions:
- Partner A: $100
- Partner B: $100
- Partner C: $100
- Partner D: $100

Σ√ci = √100 + √100 + √100 + √100 = 10 + 10 + 10 + 10 = 40
(Σ√ci)² = 40² = 1,600
Σci = 100 + 100 + 100 + 100 = 400

matched = 1,600 - 400 = $1,200

Total Funding = $400 (contributions) + $1,200 (match) = $1,600
```

**Key Insight:** Many small contributions beat one large contribution.

### Community Fund Access by Tier

| Tier | Application Priority | Match Multiplier | Voting Rights |
|------|---------------------|------------------|---------------|
| Registered | Standard | 1x | Vote on projects |
| Certified | Priority (faster review) | 1.5x | Vote + propose |

## Quadratic Funding Access

### The Matching Algorithm

When partners contribute to community projects, their contribution is matched using:

```
totalScore = (skillScore + industryBonus) × 0.6 + reputationScore × 0.4

Where:
- skillScore = overlap between partner skills and project needs (0-1)
- industryBonus = 0.1 if partner industry matches project vertical
- reputationScore = partner reputation / 1000 (capped at 1)
```

### Matching Process Flow

```
┌─────────────────────────────────────────────────────────────────┐
│  Partner contributes to community project                       │
└────────────────────┬────────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────────┐
│  Matching Algorithm Calculates Score                            │
│  ├── skillScore: 60% weight (expertise match)                  │
│  ├── industryBonus: +10% if vertical match                     │
│  └── reputationScore: 40% weight (past contributions)          │
└────────────────────┬────────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────────┐
│  Match Multiplier Applied                                       │
│  ├── Registered: 1x base                                       │
│  ├── Certified: 1.5x                                           │
│  ├── Strategic: 2x                                             │
│  └── Enterprise: Custom                                        │
└────────────────────┬────────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────────┐
│  Final Match = Base Match × Multiplier × totalScore            │
│  Funds released to project wallet                              │
└─────────────────────────────────────────────────────────────────┘
```

### Maximizing Your Match

**Tips for Partners:**

1. **Build Reputation** — Contribute consistently to high-impact projects
2. **Develop Specializations** — Higher skill scores in your vertical
3. **Collaborate** — Many small contributions > one large (quadratic!)
4. **Certify** — Higher tiers get better multipliers

### Example: Strategic Partner Match

```
Project: "Healthcare HIPAA Compliance Templates"

Partner Profile:
- Tier: Strategic (2x multiplier)
- Skills: Healthcare, Compliance, HIPAA
- Industry: HealthTech (bonus applies)
- Reputation: 850/1000

Calculation:
- skillScore: 0.9 (high overlap)
- industryBonus: 0.1 (HealthTech match)
- reputationScore: min(850/1000, 1) = 0.85

totalScore = (0.9 + 0.1) × 0.6 + 0.85 × 0.4
           = 1.0 × 0.6 + 0.34
           = 0.94

Base Match: $1,000
Final Match: $1,000 × 2.0 × 0.94 = $1,880

Partner contributes $1,000 → receives $1,880 match
```

### API Quick Start

```bash
# 1. Get your partner API key from portal
export PARTNER_API_KEY="pk_partner_abc123"

# 2. Submit mission on behalf of client
curl -X POST https://api.agencyos.network/v1/missions \
  -H "Authorization: Bearer $PARTNER_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "client_id": "client_xyz",
    "mission": "Add user authentication with OAuth2",
    "tier": "pro"
  }'

# 3. Track mission status
curl https://api.agencyos.network/v1/missions/{mission_id} \
  -H "Authorization: Bearer $PARTNER_API_KEY"
```

### Webhook Events

| Event | Description | Payload |
|-------|-------------|---------|
| `mission.started` | Mission execution began | `{mission_id, client_id, tier}` |
| `mission.completed` | Mission delivered | `{mission_id, credits_used, status}` |
| `mission.failed` | Mission failed (auto-refund) | `{mission_id, error, refund_amount}` |
| `revenue.payout` | Monthly payout processed | `{amount, period, client_count}` |

## Exit Process

### Voluntary Exit

**Notice Period:**
- Registered: 30 days
- Certified+: 60 days
- Strategic+: 90 days
- Enterprise: Per contract (typically 180 days)

**Exit Steps:**

1. **Submit Exit Notice** via partner portal or email
2. **Client Transition Plan** — Ensure no service disruption
3. **Final Revenue Reconciliation** — All payouts processed
4. **Asset Return** — Return any confidential materials, revoke access
5. **Exit Interview** — Feedback session (optional but encouraged)

**Post-Exit:**
- Revenue share continues for clients referred during partnership (perpetual, at 50% of original rate)
- Community fund access ends
- Partner badge must be removed from website

### Termination for Cause

**Grounds for Termination:**
- Fraud or misrepresentation
- Repeated brand guideline violations
- Non-payment (for paid tiers)
- Material breach of partner agreement

**Process:**
1. Written notice of breach
2. 30-day cure period
3. If uncured, immediate termination
4. No ongoing revenue share (for cause only)

### Good Leaver vs Bad Leaver

| Aspect | Good Leaver (Voluntary) | Bad Leaver (For Cause) |
|--------|------------------------|------------------------|
| **Ongoing Revenue** | 50% perpetuity | 0% |
| **Client Transition** | Supported | Immediate handoff |
| **Reference Rights** | Can mention | No mention |
| **Rejoin Eligibility** | After 12 months | Never |
