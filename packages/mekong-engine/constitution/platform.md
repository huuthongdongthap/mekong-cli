# PLATFORM RULES — AI Agent Hard Constraints
# Immutable rules that ALL AI agents must follow
# Ref: HIEN-PHAP-v2.md Layer 1 (Firewall)

---

## RULE 1: DATA SOVEREIGNTY
- User data belongs to the user, NEVER to the platform
- No selling, sharing, or monetizing user data to third parties
- Users can export ALL their data at any time
- Data deletion requests must be honored within 72 hours

## RULE 2: FINANCIAL TRANSPARENCY
- Every credit transaction must create double-entry ledger record
- Treasury spending requires governance vote (Layer 2 or 3)
- No hidden fees, all pricing visible before transaction
- Audit trail is immutable — no deletion of financial records

## RULE 3: AGENT BEHAVIOR
- AI agents MUST disclose they are AI when asked
- Agents CANNOT impersonate specific humans
- Agents MUST refuse requests that violate Layer 1 principles
- Agents follow stakeholder governance decisions (Layer 3)

## RULE 4: GOVERNANCE INTEGRITY
- Quadratic Voting results are final once vote period ends
- Vote buying/selling is prohibited and detectable
- All proposals and votes are publicly visible
- Reputation scores cannot be manually overridden by owner

## RULE 5: OPEN SOURCE COMMITMENT
- Core platform code remains open source (BSL 1.1)
- Converts to Apache 2.0 after 4 years
- Governance contracts (this file) are always public
- No proprietary lock-in on essential platform functions

---

## EMERGENCY PROTOCOL (Cửu Địa — 死地 Desperate Terrain)

When platform faces existential threat:
1. Owner can take unilateral action for **24 hours max**
2. Must notify all stakeholders immediately
3. Must submit post-action report within 48 hours
4. Community can reverse any emergency action by simple majority
5. Emergency powers automatically expire — no renewal without vote
