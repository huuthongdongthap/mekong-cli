# FOUNDER SOVEREIGNTY LAYER
# "VC ăn thịt được founder vì founder không biết luật chơi"
# Mekong CLI — Anti-dilution Intelligence System
#
# THESIS: Founder tốt nhất không cần VC. Founder thứ nhì biết đàm phán với VC.
# Founder tệ nhất ký term sheet mà không hiểu.
#
# Layer này giải quyết: information asymmetry hoàn toàn.

---

## TẠI SAO VC "ĂN THỊT" ĐƯỢC FOUNDER

```
VC có lợi thế:
  1. Thông tin bất đối xứng — VC ký 50 deals/năm, founder ký 1 lần
  2. Thời gian bất đối xứng — founder cần tiền NOW, VC không vội
  3. Luật sư bất đối xứng — VC có team legal in-house, founder thuê ngoài
  4. FOMO tactics — "offer expires Friday", "2 others interested"
  5. Complexity — term sheet 40 trang với clause founder không hiểu
  6. Social proof trap — "everybody takes this"

Kết quả điển hình:
  - Founder có 8% công ty sau Series B
  - Board bị VC control sau Series A
  - Liquidation pref = VC lấy hết nếu exit < $50M
  - Anti-dilution = founder bị dilute thêm khi round sau giá thấp
  - Drag-along = VC bán công ty, founder không cản được
```

## FOUNDER SOVEREIGNTY COMMANDS

```
LAYER 1 — INDEPENDENCE (không cần VC)
  /founder bootstrap    → Path to $1M ARR không dùng VC tiền
  /founder ramen        → Minimum viable profitability calculator

LAYER 2 — INTELLIGENCE (biết trước khi gặp VC)  
  /founder vc-map       → Map VC landscape, biết ai nào là founder-friendly
  /founder term-sheet   → Analyze term sheet, flag mọi trap
  /founder cap-table    → Model dilution scenarios trước khi ký

LAYER 3 — LEVERAGE (đàm phán từ thế mạnh)
  /founder negotiate    → Negotiation playbook, counter-term tactics
  /founder alternatives → Revenue-based financing, strategic investors, grants
  /founder governance   → Board control mechanics, protective provisions

LAYER 4 — OPTIONALITY (luôn có exit plan)
  /founder acqui-hire   → Giá trị của team nếu acquisition
  /founder profitable   → Path to profitable không cần raise thêm
```

## CORE PRINCIPLE: BATNA IS EVERYTHING

```
BATNA = Best Alternative To Negotiated Agreement

Founder có BATNA tốt = không cần VC deal xấu
Mọi command trong layer này đều build BATNA:
  /founder bootstrap → BATNA = không raise
  /founder alternatives → BATNA = raise từ nguồn khác
  /founder profitable → BATNA = không raise thêm ever

Rule: Không bao giờ vào phòng đàm phán khi BATNA = "chúng tôi cần tiền này"
```
