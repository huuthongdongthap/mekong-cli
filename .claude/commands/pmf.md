## Sean Ellis Test (Gold Standard)

```
Survey question: "How would you feel if you could no longer use [product]?"

Responses:
- Very disappointed     → Target: ≥40% for PMF
- Somewhat disappointed
- Not disappointed
- N/A (no longer use)

Result interpretation:
≥40% "very disappointed" = PMF achieved ✅
25-39% = Close, iterate on core value
<25% = Pivot or major changes needed
```

## PMF Assessment Framework

### 1. Qualitative Signals
| Signal | Pre-PMF | Post-PMF |
|--------|---------|----------|
| User acquisition | Push (outbound) | Pull (organic/referral) |
| Retention | Users churn fast | Users come back daily |
| Word of mouth | None | Organic referrals |
| Usage patterns | Sporadic | Habitual |
| Feedback | "Nice idea" | "I can't live without this" |

### 2. Quantitative Metrics
```yaml
retention:
  day1: ≥60%    # Good
  day7: ≥30%    # Strong signal
  day30: ≥15%   # PMF likely

engagement:
  dau_mau: ≥30%  # Daily/Monthly active ratio
  session_length: growing
  feature_adoption: ≥3 core features used

growth:
  organic_signup_rate: ≥40% of total
  nps_score: ≥50
  viral_coefficient: ≥0.5
```

## Superhuman PMF Framework

```
Step 1: Segment users by "very disappointed" response
Step 2: Identify what those users love most
Step 3: Find what holds back "somewhat disappointed" users
Step 4: Double down on what lovers love
Step 5: Address blockers for fence-sitters
Step 6: Re-survey → track % increase
```

## Customer Forces Canvas

```
                    PUSH (away from current)
                    ├── Frustration with status quo
                    └── Specific pain points
                    
SWITCH ──────────────────────────────── STAY
                    
                    PULL (toward new solution)          HABIT (of existing)
                    ├── New capabilities                ├── Familiar workflow
                    └── Better outcomes                 └── Switching cost
                    
                    ANXIETY (about switching)
                    ├── Will it work?
                    └── Learning curve
```

## 48-Hour PMF Sprint

| Hour | Action |
|------|--------|
| 0-4 | Deploy Sean Ellis survey to existing users |
| 4-8 | Interview top 5 "very disappointed" users |
| 8-16 | Analyze patterns — what do lovers love? |
| 16-24 | Interview top 5 "not disappointed" users |
| 24-32 | Map blockers and switching anxiety |
| 32-40 | Prioritize top 3 improvements |
| 40-48 | Create action plan with metrics |
