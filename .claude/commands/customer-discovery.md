## The Mom Test Rules

```
Rule 1: Talk about THEIR life, not YOUR idea
Rule 2: Ask about specifics in the past, not generics about the future
Rule 3: Talk less, listen more

❌ Bad: "Would you use an app that does X?"
✅ Good: "Tell me about the last time you dealt with [problem]"

❌ Bad: "Would you pay $10/month for this?"
✅ Good: "What are you currently paying to solve this problem?"

❌ Bad: "Do you think this is a good idea?"
✅ Good: "What have you tried so far to fix this?"
```

## Interview Script Template

```markdown
### Warm-up (2 min)
"Thanks for taking the time. I'm researching [topic area].
There are no right or wrong answers — I just want to learn."

### Context (5 min)
1. "Walk me through your typical day/workflow for [area]"
2. "What's the hardest part about [problem area]?"
3. "Why is that hard?"

### Problem Deep-Dive (10 min)
4. "Tell me about the last time [problem] happened"
5. "What did you do to solve it?"
6. "What didn't you like about that solution?"
7. "How much time/money does this cost you?"
8. "How often does this happen?"

### Solution Exploration (5 min)
9. "If you could wave a magic wand, what would the perfect solution look like?"
10. "What have you tried that didn't work?"

### Commitment (3 min)
11. "Would you be open to trying a solution if we built one?"
12. "Who else should I talk to about this?"
13. "Can I follow up with you in a few weeks?"
```

## Jobs-to-Be-Done (JTBD) Framework

```
When I [situation/context],
I want to [motivation/force],
So I can [desired outcome].

Example:
When I'm onboarding new clients,
I want to automate the paperwork,
So I can focus on building the relationship.
```

## Validation Signals

```
🟢 STRONG signals (proceed):
- People describe the problem with emotion
- They've already tried to solve it
- They're spending money on workarounds
- They ask "when can I use this?"

🟡 WEAK signals (iterate):
- "That sounds interesting"
- "I might use that"
- Polite but vague enthusiasm

🔴 STOP signals (pivot):
- "I don't really have that problem"
- Can't remember last time it happened
- Not willing to change current workflow
```
