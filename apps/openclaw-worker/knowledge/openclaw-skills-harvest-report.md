## 4. Operational Protocols

### cc-godmode (The Orchestrator)
- **Trigger:** `/cook "New Feature: X"` or `/cook "Bug Fix: Y"`
- **Role:** Delegates to sub-agents (`@researcher`, `@architect`, `@builder`, etc.).
- **Rule:** NEVER implements directly. Always plans and delegates.

### smart-model-switching (The Optimizer)
- **Trigger:** Automatic (embedded in prompt logic).
- **Logic:** Start with Haiku -> Escalate to Sonnet for code -> Escalate to Opus for architecture/complex bugs.
- **Goal:** Reduce token costs while maintaining quality.

### memory (The Historian)
- **Trigger:** Explicit storage/retrieval requests.
- **Storage:** `.claude/memory/` (or configured path).
- **Usage:** "Remember this architecture decision." / "Recall the fix for issue #123."

### deep-research-pro (The Scout)
- **Trigger:** `/research "topic"` or "Deep dive into X".
- **Workflow:** Plan -> Search (DDG) -> Read (Curl/Browser) -> Synthesize -> Report.
- **Output:** Cited markdown report.

### agent-browser (The Explorer)
- **Trigger:** "Go to url X and check Y".
- **Capability:** Headless browsing, clicking, typing, screenshotting.
- **Safety:** Restricted to safe domains (unless overridden).
