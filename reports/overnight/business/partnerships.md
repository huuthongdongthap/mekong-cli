# Partnership Strategy — Mekong CLI

## Overview
Partnerships accelerate Mekong CLI's distribution into developer and founder communities without paid acquisition cost. The core partnership thesis: integrate deeply with the tools developers already use, appear in the ecosystems where founders discover productivity tools, and create mutual value through co-marketing and technical integrations.

---

## Partnership Categories

### Tier 1: LLM Provider Partnerships (Technical + GTM)
**Target:** OpenRouter, DeepSeek, Qwen/DashScope, Anthropic, Ollama

**Value exchange:**
- Mekong CLI → generates LLM API usage for providers (volume referrals)
- Provider → co-marketing, featured in provider marketplace/docs
- Mutual: "Mekong CLI — recommended by OpenRouter for AI ops automation"

**Partnership mechanics:**
- Technical: Mekong CLI ships with provider preset configs (`mekong/adapters/llm-providers.yaml`)
- Co-marketing: Blog posts, newsletter mentions, Twitter shoutouts
- Revenue share: Referral links for each provider (earn on API usage referred)

**Priority order:**
1. **OpenRouter** — 300+ models, highest reach, already default in docs
2. **Qwen/DashScope** — Cheapest coding plan ($10/mo unlimited), Vietnamese market
3. **DeepSeek** — Cost-competitive, popular with bootstrapped founders
4. **Ollama** — Free local inference, drives open-source adoption

### Tier 2: Developer Tool Ecosystem (Distribution)
**Target:** Homebrew, pip/PyPI ecosystem, VS Code extensions, Raycast

**Value exchange:**
- Mekong CLI listed in tool ecosystems → passive discovery
- Ecosystem promotion → legitimacy and trust signal

**Specific targets:**
- **Homebrew**: `brew install mekong-cli` — Mac developer default install method
- **PyPI**: Already on pip (`pip install mekong-cli`) — maintain and promote
- **Raycast**: Mekong CLI extension for quick command execution from Raycast bar
- **VS Code**: Extension to run `mekong cook` from command palette
- **Warp terminal**: Featured in Warp's AI tools directory

### Tier 3: Community & Platform Partnerships (Audience)
**Target:** IndieHackers, Hacker News, Dev.to, Y Combinator community

**Value exchange:**
- Sponsored posts / community presence → developer audience exposure
- Case studies featuring their community members using Mekong CLI

**Specific targets:**
- **IndieHackers**: Sponsored product post + founder testimonials
- **Y Combinator**: YC founder community — Mekong CLI as "ops layer for YC startups"
- **Dev.to organizations**: Mekong CLI organization account with team posts
- **Cloudflare blog**: Co-authored technical post (Mekong + CF Workers + D1)

## Affiliate Program (command: `mekong affiliate`)

### Program Structure
| Tier | Requirement | Commission | Duration |
|------|------------|-----------|----------|
| Standard affiliate | Any signup | 20% of first 12 months | 12 months |
| Power affiliate | 10+ referrals/mo | 25% + featured listing | 12 months |
| Agency partner | White-label resell | 30% recurring | Lifetime |

### Affiliate Mechanics
- Unique referral link per affiliate (tracked via Polar.sh)
- 30-day cookie window
- Monthly payout via Polar.sh (on successful retention, not just signup)
- Affiliate dashboard: conversions, earnings, link performance

### Target Affiliates
1. **Developer YouTubers**: Tools-focused channels (Theo, Fireship, Traversy) — $500-1000 sponsorship for initial video
2. **Newsletter authors**: Developer/founder newsletters (TLDR, Pointer, JavaScript Weekly)
3. **AI tools review sites**: There's an AI, FutureTools, Toolify
4. **Course creators**: Python/AI course instructors who can embed Mekong CLI in curriculum
5. **Existing users**: Power users with Twitter/GitHub followings

### Affiliate Workflow (`mekong affiliate`)
```
1. mekong affiliate/scout   → Find high-potential affiliates in target channels
2. mekong affiliate/compare → Evaluate affiliate program ROI vs direct marketing
3. Outreach + agreement     → Standard affiliate agreement template
4. Onboarding              → Affiliate kit: demo video, copy templates, link
5. Monthly review          → Performance vs payout, tier upgrades
```

## Proposal Process (command: `mekong proposal`)

### Partnership Proposal Template (generated by `mekong proposal`)
Structure for approaching any partner:

1. **The ask**: What Mekong CLI wants (co-marketing, integration, listing)
2. **Their win**: What partner gets (new product feature, audience engagement, revenue)
3. **The proof**: GitHub stars, install count, customer testimonials
4. **The mechanics**: How it works technically and commercially
5. **The numbers**: Estimated reach, conversion, revenue share

### Proposal Prioritization
| Partner Type | Value | Effort | Priority |
|-------------|-------|--------|----------|
| OpenRouter co-marketing | High | Low | Immediate |
| Homebrew listing | Medium | Low | Week 1 |
| Cloudflare blog post | High | Medium | Month 1 |
| Raycast extension | Medium | High | Month 2 |
| YC community deal | Very High | High | Month 3 |

## Partnership Management

### Partnership CRM (tracked via `mekong crm` + `mekong partnerships`)
- Partner name, tier, contact
- Agreement status: prospecting → proposal sent → active → renewing
- Co-marketing calendar: upcoming joint posts, campaigns
- Revenue attributed: MRR traced to partner channel

### Partnership Review Cadence
- Monthly: Review active partner performance (traffic, signups, revenue)
- Quarterly: Renegotiate or expand high-performing partnerships
- `mekong business-quarterly-review` includes partner channel attribution

## Partnership Risk & Mitigation

| Risk | Mitigation |
|------|-----------|
| LLM provider dependency | Multi-provider by design — no single provider lock-in |
| Affiliate fraud | Minimum 30-day retention before payout |
| Partner churning | Diversify across 20+ affiliates |
| Co-marketing promises | Contractual commitment with `mekong agreement` |
