## Credit System Explained

### What is 1 Credit?

**1 Credit = 1 MCU (Mission Control Unit)**

Credits are the currency for mission execution. Each mission's complexity determines its cost.

### Mission Complexity Tiers

| Complexity | Credits | Character Length | Example |
|------------|---------|------------------|---------|
| **Simple** | 1 | < 50 chars | "Fix typo in README" |
| **Standard** | 3 | 50-149 chars | "Add user authentication" |
| **Complex** | 5 | ≥ 150 chars | "Migrate database schema" |
| **Multi-Agent** | 10+ | Parallel orchestration | "Full sprint automation" |

### Auto-Detection

Complexity is automatically detected from mission length:
- Simple: < 50 characters
- Standard: 50-149 characters
- Complex: ≥ 150 characters

### Unused Credits

- **Starter:** No rollover
- **Pro:** Rollover enabled
- **Enterprise:** Unlimited (no tracking needed)

## Quality Gates — Binh Phap Standards

Every mission MUST pass these non-negotiable quality gates before delivery:

### Six Quality Fronts

| Front | Target | Verification |
|-------|--------|--------------|
| **Tech Debt** | 0 TODO/FIXME, 0 console.log | `grep -r "TODO\|console\." src` |
| **Type Safety** | 0 `any` types | `grep -r ": any" src` |
| **Performance** | Build < 10s, LCP < 2.5s | `time npm run build` |
| **Security** | Input validation, no secrets | `npm audit`, secret scanning |
| **UX** | Loading states, error boundaries | Manual review |
| **Documentation** | Self-documenting code | README, inline comments |

### Automated Enforcement

```

   Type safety enforced — no lazy typing

✅ 0 `TODO`/`FIXME` comments
   No tech debt left behind

✅ 0 `console.log` in production

✅ Build < 10s required
```
